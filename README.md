# 임상연구 데이터 품질관리 자동화 시스템
**Clinical Research Data Quality Management System**

---

## 개발 배경

병원 내 IRB(기관생명윤리위원회) 및 DRB(데이터심의위원회) 승인 기반 임상연구에서는  
병록번호를 기준으로 연구ID(R-ID)를 부여받아 데이터를 관리합니다.

그러나 실제 운영 과정에서 두 가지 반복적인 문제가 발생했습니다.

1. **R-ID 추출 프로그램 오류** — 외부 시스템에서 R-ID를 추출하는 과정에서 파싱 오류로 인한 데이터 누락·오기 발생
2. **수기 입력 휴먼에러** — 수집일지를 엑셀에 직접 입력하는 과정에서 성별·생년월일 오기입 반복 발생

이로 인해 연구 데이터 분양 및 코호트 구축 시 데이터 불일치 문제가 지속되었고,  
수동 검토에 과도한 시간이 소요되어 **자동화 시스템 개발**을 진행하였습니다.

---

## 주요 기능

### 1. Master DB 구축
- 다수의 수집일지(엑셀) 파일을 병합하여 통합 Master DB 생성
- 같은 병록번호 내 식별번호 우선순위 자동 적용 (O식별번호 > X식별번호)
- 가족/보호자 데이터 자동 분류 (가족 키워드 기반)
- 오기 데이터 자동 탐지 및 분류 (오기데이터 시트 별도 관리)
- bCODE·식별번호 변경 이력 자동 기록

### 2. 병록번호·접수일자 추출
- Master DB에서 병록번호와 접수일자를 추출하여 R-ID 요청용 파일 생성
- 외부 시스템(슈프림 등) 제출 포맷에 맞게 자동 변환

### 3. 가명화 데이터 매칭 (R-ID 매칭)
- IRB·DRB 승인 하에 병록번호 기준으로 수령한 R-ID 파일과 수집 데이터를 자동 매칭
- 생성ID(성별+생년월일+이름+접수일자 기반 내부 식별키)를 이용한 정합성 검증
- 매칭 성공 / 미매칭 / 중복 자동 분류
- 미매칭 데이터에서 슈프림 확인용 파일 자동 생성

### 4. 오기입 보정 매칭 (Hubis 데이터 기반)
- 외부 데이터소스(Hubis) 기준으로 성별·생년월일 오기입 자동 탐지 및 보정
- 보정 후 재매칭 수행
- 비밀번호 보호된 R-ID 파일 직접 복호화 처리 (msoffcrypto 활용)

## 데이터 흐름

```
수집일지 엑셀 파일들
        │
        ▼
  [1단계 전처리]
  중복 제거 / 식별번호 우선순위 / 가족 분류 / 오기 탐지
        │
        ▼
   Master DB 생성
  (매칭파일 / 오기데이터 / 가족데이터 / HISTORY)
        │
        ▼
  병록번호+접수일자 추출 → 외부 시스템에 R-ID 요청
        │
        ▼
   R-ID 파일 수령 (IRB·DRB 승인 기반)
        │
        ▼
  [가명화 데이터 매칭]
  생성ID 역산 → R-ID 매칭 → 매칭성공 / 미매칭 / 중복 분류
        │
        ▼
  [Hubis 오기입 보정 매칭]
  외부 데이터 기준 성별·생년월 보정 → 재매칭
        │
        ▼
  최종 매칭파일 (데이터 분양 / 코호트 구축)
```

## 기술 스택

| 항목 | 내용 |
|---|---|
| Language | Python 3.x |
| GUI | Tkinter |
| 데이터 처리 | pandas, numpy |
| 엑셀 입출력 | openpyxl, python-calamine |
| 암호화 파일 처리 | msoffcrypto-tool |

---

## 실행 방법

```bash
python 매칭파일\ 생성.py
```

실행 시 GUI 창이 열리며 아래 4가지 기능을 선택할 수 있습니다.

| 버튼 | 기능 |
|---|---|
| 1. DB 생성 | 수집일지 파일들을 병합하여 Master DB 생성 |
| 2. 병록번호, 접수일자 추출 | Master DB에서 R-ID 요청용 파일 추출 |
| 3. 가명화매칭 | 수령한 R-ID 파일과 Master DB 매칭 |
| 4. 매칭실패데이터 Hubis 매칭 | 오기입 보정 후 재매칭 |

---

## 샘플 데이터

`sample_data/` 폴더에 가상 데이터 파일이 포함되어 있습니다.  
실제 환자 데이터는 포함되어 있지 않으며, 모든 샘플 데이터는 `generate_sample_data.py`로 생성된 가상 데이터입니다.

## 개선 효과

- 수동 검토 대비 데이터 정합성 검증 시간 단축
- 오기입 데이터 자동 탐지로 휴먼에러 최소화
- 변경 이력 자동 관리로 데이터 추적성 확보
- 데이터 분양 및 코호트 구축 품질 향상
