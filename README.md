# 임상연구 데이터 품질관리 자동화 시스템
**Clinical Research Data Quality Management System**

---

## 개발 배경

병원 내 IRB·DRB 승인 기반 임상연구에서는 연구별 고유 ID(R-ID)를 기준으로 데이터를 관리합니다.

실제 운영 과정에서 아래 세 가지 문제가 반복적으로 발생했습니다.

1. **수기 입력 휴먼에러** — 연구원이 검체를 보고 엑셀에 직접 기록하는 과정에서 성별·생년월일 오기입 반복 발생
2. **외부 시스템 데이터 불일치** — 데이터를 보관·활용하는 두 외부 시스템 모두 오류가 존재하여 교차 검증 필요
3. **수정 후 재반영 미흡** — 오류 발견·수정 완료 후 원본 수집일지에 반영하는 과정이 수동

이로 인해 데이터 분양 및 코호트 구축 시 품질 문제가 지속되고 수동 검토 시간이 과도하게 소요되어 **자동화 시스템 개발**을 진행하였습니다.

---

## 주요 기능

### 1. Master DB 구축
- 다수의 수집일지(엑셀) 파일을 병합하여 통합 Master DB 생성
- 같은 병록번호 내 식별번호 우선순위 자동 적용 (P식별번호 > F식별번호)
- 가족·보호자 데이터 자동 분류 (키워드 기반)
- 이상 데이터 자동 탐지 및 분류 (오기데이터 시트 별도 관리)

### 2. 병록번호·접수일자 추출
- Master DB에서 병록번호와 접수일자를 추출하여 R-ID 요청용 파일 생성

### 3. R-ID 매칭
- IRB·DRB 승인 하에 수령한 R-ID 파일과 Master DB를 자동 매칭
- 매칭 성공 / 미매칭 / 중복 자동 분류
- 결과 파일에 R-ID 컬럼은 매칭성공 시트에만 포함 (미매칭·중복 시트 제외)

### 4. 오기입 보정 매칭
- 외부 데이터소스 기준으로 성별·생년월일 오기입 자동 탐지 및 보정
- 보정 후 재매칭 수행
- 보정 건수·재매칭 결과 완료 메시지에 표시

### 5. 오류 알림 발송
- 미매칭 데이터 발생 시 담당자에게 이메일로 자동 알림
- 수집관리시스템 담당자: 미매칭 내부코드 목록 발송
- 데이터추출시스템 담당자: 미매칭 병록번호 목록 발송
- 발송 전 미리보기 창으로 내용 확인 후 전송

### 6. 수집일지 자동 수정
- 담당자로부터 수정 완료 데이터 수령 시 원본 수집일지에 자동 반영
- 수집관리시스템: 내부코드 기준 매칭 → 개인번호(성별·생년월) 자동 업데이트
- 데이터추출시스템: 병록번호 기준 매칭 → 변경된 항목만 덮어쓰기
- 원본 파일과 동일한 시트 구조·포맷으로 출력

## 데이터 흐름

```
수집일지 엑셀 파일들
        │
        ▼
  [1단계: Master DB 생성]
  병합 / 식별번호 우선순위 / 가족 분류 / 이상 탐지
        │
        ▼
   Master_DB.xlsx
   (통합데이터 / 오기데이터 / 가족데이터 / 변경이력)
        │
        ▼
  [2단계: 병록번호·접수일자 추출]
  외부 시스템에 R-ID 요청
        │
        ▼
   R-ID 파일 수령 (IRB·DRB 승인 기반)
        │
        ▼
  [3단계: R-ID 매칭]
  매칭성공 / 미매칭 / 중복 분류
        │
   ┌────┴──────────────┐
   │                   │
  매칭성공            미매칭
   │                   │
   ▼                   ▼
  완료         [5단계: 오류 알림 발송]
               미매칭 목록 → 이메일 → 담당자 수정 요청
                        │
                        ▼
               [4단계: 오기입 보정 매칭]
               외부 데이터소스 기준 보정 → 재매칭
                        │
                        ▼
               [6단계: 수집일지 자동 수정]
               수정 완료 데이터 수령 → 원본 수집일지 업데이트
```

## 실행 방법

```bash
python clinical_data_qc.py
```

실행 시 GUI 창이 열리며 아래 6가지 기능을 선택할 수 있습니다.

| 버튼 | 기능 |
|---|---|
| 1. Master DB 생성 | 수집일지 파일들을 병합하여 Master DB 생성 |
| 2. 병록번호·접수일자 추출 | Master DB에서 R-ID 요청용 파일 추출 |
| 3. R-ID 매칭 | 수령한 R-ID 파일과 Master DB 매칭 |
| 4. 오기입 보정 매칭 | 외부 데이터소스 기준 보정 후 재매칭 |
| 5. 오류 알림 발송 | 미매칭 목록을 담당자에게 이메일 발송 |
| 6. 수집일지 자동 수정 | 수정 완료 데이터 수령 후 원본 수집일지 업데이트 |

## 샘플 데이터

`sample_data/` 폴더에 가상 데이터 파일이 포함되어 있습니다.
실제 환자 데이터는 포함되어 있지 않으며, 모든 샘플 데이터는 `generate_sample_data.py`로 생성된 가상 데이터입니다.

| 파일 | 사용 기능 |
|---|---|
| `수집일지_샘플.xlsx` | 1번 (Master DB 생성), 6번 (수집일지 자동 수정) |
| `Master_DB_샘플.xlsx` | 2번 (병록번호 추출), 3번 (R-ID 매칭) |
| `rid_파일_샘플.xlsx` | 3번 (R-ID 매칭), 4번 (보정 매칭) |
| `수집관리시스템_샘플.xlsx` | 4번 (보정 매칭), 6번 (수집관리시스템 수정파일) |
| `데이터추출시스템_수정파일_샘플.xlsx` | 6번 (데이터추출시스템 수정파일) |

> 3번 기능 실행 후 생성되는 `매칭결과.xlsx`는 5번·4번의 입력 파일로 사용됩니다.

## 품질 관리 체계

### AI 에이전트 하네스 (Agent Harness)

AI가 이 프로젝트를 수정할 때 따라야 할 행동 원칙과 컨텍스트를 문서로 정의합니다.

| 문서 | 역할 |
|---|---|
| `CLAUDE.md` | 핵심 원칙 요약 및 코드 위치 참조 |
| `AGENTS.md` | AI 행동 원칙 (절대 금지·항상 유지·커밋 원칙) |
| `ARCHITECTURE.md` | 6단계 파이프라인 구조 및 데이터 흐름 |
| `docs/design-docs/` | 절대 변경 불가 핵심 원칙, 데이터 모델 |
| `docs/product-specs/` | 6개 기능별 입력·처리·출력 스펙 |
| `docs/exec-plans/` | 실행 계획 및 기술 부채 관리 |
| `docs/QUALITY_SCORE.md` | 배포 가능 기준 체크리스트 |
| `docs/RELIABILITY.md` | 파일·데이터·GUI 안정성 원칙 |
| `docs/SECURITY.md` | 환자 데이터·이메일 보안 원칙 |

### 테스트 하네스 (Test Harness)

GUI·파일 I/O 없이 순수 함수만 단위 테스트합니다.

```bash
pytest tests/
```

| 테스트 대상 | 검증 내용 |
|---|---|
| `clean_str` | NaN 처리, 공백 제거, `.0` 후처리 |
| `pad_hosp_num` | 7자리 → 8자리 패딩 |
| `resolve_year` | 2자리 연도 → 4자리 변환 |
| `parse_personal_no` | 개인번호(M93.05) 파싱 |
| `is_family_name` | 가족 키워드 감지 |
| `_detect_columns` | R-ID 파일 컬럼 자동 감지 |
| `_build_email_body` | 이메일 본문 생성 |

**39개 테스트 전체 통과**

### 린터 (Linter)

```bash
ruff check .
```

`ruff.toml` 기준으로 E·W·F 규칙을 적용합니다. **오류 0개** 상태를 유지합니다.

## 개선 효과

- 수기 입력 오류 자동 탐지로 휴먼에러 최소화
- 미매칭 발생 즉시 담당자 자동 통보로 오류 대응 속도 향상
- 수정 완료 데이터 수령 후 원본 반영 자동화로 수작업 제거
- 데이터 분양 및 코호트 구축 품질 향상
