# AGENTS.md — AI 에이전트 행동 원칙

이 파일은 Claude 또는 다른 AI 에이전트가 이 프로젝트를 수정할 때 반드시 따라야 할 규칙을 정의합니다.

---

## 필수 참조 순서

코드를 수정하기 전 반드시 아래 순서로 문서를 확인합니다.

1. `ARCHITECTURE.md` — 전체 구조 파악
2. `docs/design-docs/core-beliefs.md` — 절대 변경 불가 원칙 확인
3. 해당 기능의 `docs/product-specs/` 문서 확인
4. `docs/QUALITY_SCORE.md` — 품질 기준 충족 여부 확인

---

## 코드 수정 원칙

### 절대 금지
- `COL_*` 상수를 하드코딩된 문자열로 교체하는 행위
- 식별번호 우선순위(O > X > 기타) 로직 제거 또는 변경
- 생성ID 생성 규칙 변경 (성별+출생연도+출생월+이름첫글자+접수일자)
- R-ID 컬럼을 미매칭·중복 시트에 포함시키는 행위
- 오류 알림 기능에서 실제 SMTP 발송 코드 추가 (데모 환경 유지)

### 항상 유지
- 병록번호는 항상 8자리 (7자리 입력 시 앞에 0 추가)
- 모든 기능에 `ProgressWindow` 진행 상태창 사용
- 파일 선택은 반드시 tkinter 다이얼로그 사용
- 결과 파일은 입력 파일과 동일 폴더에 저장
- 오류 발생 시 `messagebox.showerror`로 사용자에게 안내

### 기능 추가 시
- `docs/product-specs/`에 스펙 문서를 먼저 작성
- `docs/exec-plans/active/`에 실행 계획 추가
- 완료 후 `docs/exec-plans/completed/`로 이동

## 테스트 원칙

- 실제 환자 데이터 없이 `sample_data/`의 가상 데이터로만 테스트
- 순수 함수 변경 시 반드시 `pytest tests/` 실행하여 통과 확인
- GUI 기능은 실행하여 직접 확인
- 엑셀 출력 파일의 시트 구조와 컬럼 순서 반드시 검증

---

## 린터 원칙

- 코드 수정 후 반드시 `ruff check .` 실행
- 오류 0개 상태 유지
- `ruff.toml`의 ignore 규칙을 임의로 추가하지 않음

---

## 커밋 원칙

- 실제 데이터 파일(*.xlsx, *.xlsm, *.csv)은 절대 커밋 금지
- `sample_data/` 폴더의 가상 데이터만 허용
- 커밋 메시지는 변경된 기능 단위로 명확하게 작성
