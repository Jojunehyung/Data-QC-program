# Quality Score — 품질 기준

## 코드 품질 체크리스트

### 필수 (모두 통과해야 배포 가능)
- [ ] 컬럼명 하드코딩 없음 (COL_* 상수 사용)
- [ ] 모든 기능에 ProgressWindow 진행 상태창 존재
- [ ] 파일 선택은 tkinter 다이얼로그 사용
- [ ] 결과 파일은 입력과 같은 폴더에 저장
- [ ] 오류 시 messagebox.showerror 안내
- [ ] 미매칭·중복 시트에 R-ID 컬럼 없음
- [ ] 매칭결과 모든 시트에 생성ID 컬럼 없음
- [ ] 접수일자 누락·미래날짜 행은 오기데이터 시트로 분류
- [ ] 병록번호 텍스트 서식 적용
- [ ] `ruff check .` 오류 0개
- [ ] `pytest tests/` 전체 통과

### 권장
- [ ] 함수 단위 분리 명확
- [ ] 예외 처리 후 traceback 출력
- [ ] 매직넘버 상수화

## 데이터 품질 기준

| 기준 | 허용 범위 |
|---|---|
| 병록번호 길이 | 8자리 숫자 (7자리는 자동 패딩) |
| 생성ID | 내부 로직에 의해 자동 생성 (비어있지 않은 문자열) |
| 접수일자 | 날짜 파싱 가능한 형식 |
| 내부코드 | 숫자 문자열 |
