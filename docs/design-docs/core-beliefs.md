# Core Beliefs — 절대 변경 불가 핵심 원칙

이 문서의 원칙은 어떤 이유로도 변경·우회할 수 없습니다.

---

## 1. 식별번호 우선순위

같은 병록번호 내 여러 식별번호가 존재할 때:

```
O식별번호 (환자 본인)  >  X식별번호 (가족/보호자)  >  기타
```

- **O식별번호가 있을 때**: O가 메인, X는 전부 가족데이터
- **O식별번호가 없을 때**: 가장 최신 X가 메인, 나머지 X는 가족데이터

---

## 2. 생성ID 생성 규칙

```
생성ID = 성별코드 + 출생연월(연도4자리+월2자리) + 이름첫글자 + 접수일자(YYYYMMDD)
```

예시: `M199305김20230415`

- 성별코드: `M` 또는 `F`
- 출생연월 2자리 연도 변환: `YEAR_THRESHOLD`(24) 이하 → 2000년대, 초과 → 1900년대
- 이 규칙은 R-ID 매칭 로직과 반드시 동일해야 함

---

## 3. R-ID 컬럼 노출 금지

- 미매칭 시트: R-ID 컬럼 제거
- 중복매칭 시트: R-ID 컬럼 제거
- 매칭성공 시트: R-ID 포함

---

## 4. 병록번호 정규화

- 7자리 병록번호는 반드시 앞에 `0`을 추가하여 8자리로 통일
- 엑셀 저장 시 병록번호 컬럼은 텍스트 서식 적용

---

## 5. 데이터 보존 원칙

- 처리 중 오류 발생 시 원본 파일 변경 없음
- 오류 데이터는 버리지 않고 오기데이터·가족데이터 시트로 분류

---

## 6. 오류 알림 데모 원칙

- 실제 SMTP 발송은 구현하지 않음
- 미리보기 창으로 발송 내용 확인까지만 제공
- 실제 환경 연결 코드는 주석으로만 표시
