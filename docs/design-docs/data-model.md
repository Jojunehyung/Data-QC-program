# Data Model — 데이터 구조 정의

## 컬럼 상수 정의

| 상수명 | 컬럼명 | 형식 | 설명 |
|---|---|---|---|
| `COL_BCODE` | bCODE | 숫자 문자열 | 기관 내 환자 고유 코드 |
| `COL_ID_NO` | 식별번호 | O/X + 숫자 | 환자(O) 또는 가족(X) 식별자 |
| `COL_HOSP_NUM` | 병록번호 | 8자리 숫자 문자열 | 입원 기록 번호 (7자리 입력 시 0 추가) |
| `COL_NAME` | 이름 | 한글 | 환자 이름 |
| `COL_PERS_NO` | 개인번호 | M93.05 또는 F93.05 | 성별 + 출생연월 |
| `COL_DATE` | 접수일자 | YYYY-MM-DD | 데이터 수집일 |
| `COL_GEN_ID` | 생성ID | 문자열 | 내부 식별키 (자동 생성) |
| `COL_RID` | R-ID | 문자열 | IRB 승인 기반 연구 ID |

## Master_DB.xlsx 시트 구조

### 통합데이터 (메인 시트)
```
bCODE | 식별번호 | 병록번호 | 이름 | 개인번호 | 접수일자 | 생성ID
```

### 오기데이터
```
bCODE | 식별번호 | 병록번호 | 이름 | 개인번호 | 접수일자 | 생성ID | 비고
```

### 가족데이터
```
bCODE | 식별번호 | 병록번호 | 이름 | 개인번호 | 접수일자 | 생성ID | 비고
```

### 변경이력
```
병록번호 | 이름 | 구분 | bCODE | 식별번호 | 접수일자
```

## 매칭결과.xlsx 시트 구조

| 시트 | R-ID 포함 | 내용 |
|---|---|---|
| 매칭성공 | ✅ | 매칭 성공 데이터 |
| 미매칭 | ❌ | R-ID 매칭 실패 |
| 중복 | ❌ | 생성ID 중복으로 매칭 불가 |

## 슈프림 수정파일 형식

- 시트명: `슈프림 추출 데이터(프로토콜번호)` 형식

- 성별·생년월일·환자명 컬럼 포함
