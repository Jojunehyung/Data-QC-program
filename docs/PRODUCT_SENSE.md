# PRODUCT_SENSE.md — 제품 관점 정의

## 이 프로그램이 해결하는 문제

임상연구 데이터 수집 현장에서 반복 발생하는 구조적 문제:

1. **수기 입력 휴먼에러** — 연구원이 검체를 보고 엑셀에 직접 기록하는 과정에서 성별·생년월일 오기입 반복 발생
2. **외부 시스템 데이터 불일치** — 휴비스쌤·슈프림 두 시스템 모두 오류가 있어 교차 검증이 필요하나 수동 작업에 과도한 시간 소요
3. **수정 후 재반영 미흡** — 오류를 발견해 수정 요청해도 원본 수집일지에 반영하는 과정이 수동

## 핵심 사용자

- 임상연구 데이터 관리 담당자
- 매일 수집일지를 직접 작성하고 관리하는 실무자
- IT 전문가가 아닌 환경에서 사용

## 성공 기준

- 미매칭 건수가 이전 대비 감소
- 오류 발견 후 담당자 통보까지 소요 시간 단축
- 수정파일 수령 후 원본 반영 자동화로 수작업 제거

## 비타협 원칙

| 원칙 | 이유 |
|---|---|
| 데이터 유실 없음 | 원본 파일 변경 불가, 오류 데이터 반드시 분류 보존 |
| 단일 파일 실행 | 비IT 실무자 환경 고려 |
| GUI 기반 | 커맨드라인 사용이 어려운 환경 |
| 데모 모드 유지 | 포트폴리오 배포 시 실제 환자 데이터·메일 서버 노출 방지 |

## 사용자 경험 원칙

- 버튼 하나로 기능이 시작되어야 함
- 처리 중 진행 상황이 항상 표시되어야 함 (ProgressWindow)
- 오류 메시지는 원인을 알 수 있게 구체적으로
- 결과 파일은 입력 파일 옆에 바로 생성
