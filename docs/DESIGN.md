# DESIGN.md — 설계 철학

## 핵심 설계 원칙

**1. 사용자 개입 최소화**
모든 파일 선택 외 처리는 완전 자동화.
사용자는 파일을 고르고 버튼만 누르면 된다.

**2. 데이터 유실 없음**
처리 중 오류 발생 시 원본 파일을 변경하지 않는다.
오류 데이터는 버리지 않고 분류하여 보존한다.

**3. 오류는 분류하지 버리지 않는다**
처리할 수 없는 데이터는 오기데이터·가족데이터·중복 시트로 분류.
비고 컬럼에 사유를 명시하고 추적 가능하게 유지한다.

**4. 단일 파일 실행**
의존성 설치 후 Python 파일 하나만 실행하면 동작.
복잡한 설치 과정 없음.

**5. 데모와 실제 환경 분리**
오류 알림 발송 등 외부 연동 기능은 데모 모드로 동작.
실제 연결 코드는 주석으로만 표시하여 포트폴리오 안전성 유지.
