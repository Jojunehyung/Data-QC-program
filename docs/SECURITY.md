# Security — 보안 원칙

## 데이터 보안

- 실제 환자 데이터 파일(*.xlsx, *.xlsm, *.csv)은 절대 버전 관리(git)에 포함 금지
- `sample_data/` 폴더의 가상 데이터만 허용
- 실제 내부코드, 병록번호, 환자 이름이 코드에 하드코딩되어 있지 않은지 확인

## 이메일 보안

- 실제 SMTP 자격증명(아이디·비밀번호)을 코드에 포함 금지
- 데모 환경에서는 발송 없이 미리보기만 제공
- 실제 수신자 이메일 주소를 샘플 데이터나 코드에 포함 금지

## 접근 제어

- 파일 선택은 사용자가 직접 다이얼로그에서 선택 (경로 하드코딩 금지)
- 처리 결과는 입력 파일 폴더에만 저장 (외부 경로 금지)

## 포트폴리오 보안 조치

이 저장소는 공개 포트폴리오용으로, 실제 운영 환경의 정보가 외부에 노출되지 않도록 아래 조치를 적용하였습니다.

- 실제 외부 시스템명은 기능 기반의 가상 명칭으로 대체
- 내부 환자 코드 명칭은 일반적인 가상 명칭으로 대체
- 내부 분류 체계(식별번호 접두사)는 중립적인 기호로 대체
- 내부 식별키 생성 알고리즘 형식은 공개하지 않고 내부 로직으로 처리
- 실제 DB 테이블·필드명은 가상 명칭으로 대체
- 실제 IRB/DRB 프로토콜 번호는 데모용 번호로 대체
- 매칭결과 출력 파일에서 내부 식별키 컬럼 제거
- 샘플 파일명은 가상 시스템명 기반으로 작성
