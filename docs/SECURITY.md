# Security — 보안 원칙

## 데이터 보안
- 실제 환자 데이터 파일(*.xlsx, *.xlsm, *.csv)은 절대 버전 관리(git)에 포함 금지
- `sample_data/` 폴더의 가상 데이터만 허용
- 실제 내부코드, 병록번호, 환자 이름이 코드에 하드코딩되어 있지 않은지 확인

## 이메일 보안
- 실제 SMTP 자격증명(아이디·비밀번호)을 코드에 포함 금지
- 데모 환경에서는 발송 없이 미리보기만 제공
- 실제 수신자 이메일 주소를 샘플 데이터나 코드에 포함 금지

## 접근 제어
- 파일 선택은 사용자가 직접 다이얼로그에서 선택 (경로 하드코딩 금지)
- 처리 결과는 입력 파일 폴더에만 저장 (외부 경로 금지)

## 외부 노출 방지를 위한 명칭 대체

포트폴리오 공개 시 실제 내부 시스템·용어 노출을 방지하기 위해 아래 명칭을 대체하였습니다.

| 원래 명칭 | 대체 명칭 | 이유 |
|---|---|---|
| 휴비스쌤 | 수집관리시스템 | 실제 외부 데이터관리 시스템명 |
| 슈프림 | 데이터추출시스템 | 실제 EMR 연계 데이터 추출 프로그램명 |
| bCODE | 내부코드 | 기관 내부 환자 코드 명칭 |
| O-식별번호 / X-식별번호 | P-식별번호 / F-식별번호 | 내부 환자/가족 분류 체계 노출 방지 |
| `_build_rid_map()` | `_build_match_map()` | R-ID 매칭 메커니즘 노출 방지 |
| `_read_hubis_fix()` | `_read_datasource_fix()` | 시스템명 노출 방지 |
| `_read_supreme_fix()` | `_read_emr_fix()` | 시스템명 노출 방지 |
| 생성ID 형식 공개 | 내부 로직 처리 | 내부 키 생성 알고리즘 노출 방지 |
| mg_donor_info:* 컬럼명 | donor_code, sex, birth_year, birth_month | 실제 DB 테이블·필드명 노출 방지 |
| 1004-037-315 | DEMO-000-000 | 실제 IRB/DRB 프로토콜 번호 노출 방지 |
| 매칭결과 파일 내 생성ID 컬럼 | 제거 | 내부 식별키 값에서 형식 노출 방지 |
