# Reliability — 안정성 원칙

## 파일 처리 안정성
- 개별 파일 읽기 오류는 건너뛰고 나머지 파일로 계속 진행
- 필수 컬럼 누락 시 사용자에게 구체적인 오류 메시지 표시
- 수정파일에서 병록번호를 찾지 못하면 처리 중단 후 안내

## 데이터 안정성
- 수집일지 자동 수정 시 변경된 값만 덮어쓰기 (누락값은 건드리지 않음)
- 원본 시트 구조·컬럼 순서 그대로 출력
- 병록번호 텍스트 서식 항상 유지

## GUI 안정성
- 진행 상태창(ProgressWindow)으로 처리 현황 실시간 안내
- 모든 예외는 messagebox.showerror로 사용자에게 전달
- Toplevel 창 닫힘 시 TclError 방어 처리
